## 环境兼容性限制（已知问题）

### WhisperX 在 CPU + Python 3.13 + torch 2.8 上的阻塞

| 层级 | 问题 | 详情 | 影响 |
|------|------|------|------|
| torch 2.8 | `weights_only=True` 默认值改变 | pyannote VAD 模型用 omegaconf.ListConfig pickle，被 `weights_only=True` 拒绝 | WhisperX VAD 加载失败 |
| speechbrain 1.1.0 | 引入 `k2-fsa` 懒加载 | `k2` 包的 C++ 扩展 `_k2` 在 Windows CPU 上无预编译 wheel | ImportError 级联 |
| WhisperX + CPU | medium 模型即使加载成功 | float16 回退 float32，13s 推理产出**空转录** | align soft-fail → embedding_ratio=0% |

**实测验证**：`faster-whisper small` 在 CPU 上 1.9s 转完 60s 中文音频，精确产出 18 segments。瓶颈不在推理模型，而在 WhisperX 的 VAD/依赖链初始化。

**解方**（需二选一）：
1. GPU 环境：float16 正常，medium 模型中文转录准确率高 → embedding_ratio > 0%
2. CPU 环境：改 WhisperX 为 faster-whisper 后端，或用 `whisperx_model: small/tiny`（需先解决 pyannote VAD 加载问题）

### whisperx 与 faster-whisper 的后端差异

当前 pipeline `align.py` 使用 WhisperX API（`whisperx.load_model` → `whisperx.align`）。faster-whisper 是 CTranslate2 实现的纯推理引擎，输出格式兼容 WhisperX 但 API 不同。如需替换：
- `align.py` 的 `load_model` / `align` 调用需改为 faster-whisper 接口
- 或注册 faster-whisper 作为 WhisperX 的 ASR 后端（社区未实现）

## 关键发见

1. **PR #67 是解开 O10 的关键**。仅 PR #65（仅修 align 路径）不够——match.py 的 `_transcribe_video_audio` 仍然默认走 WhisperX。在 Windows CPU 上，WhisperX cast 失败产生 float16 报错，返回 None→全部 placeholder label→ fake_captions 串发。faster-whisper fallback 接手后，假 label 下降从 100% 到 ~25%，embedding 路径激活。

2. **横向一致性**。古裝历史片（满江红）与现代喜剧片（飞驰人生3）同样达成 `embedding_ratio = 1.0`。这不是一次“优良运气”——跨题材、跨说话密度（3 vs 35 segments）、跨场景量级（2837 vs 2424 scenes）都有效。证实 PR #67 在该 pipeline 上是结构性修复，不为说话量所限。

3. **O10 解锁是“嵌入”式**。`embedding_ratio = 全部 18 segments 走 embedding`不取决于 scenes 数或 ASR 密度——只要任何 scenes 拿到过 dialogue caption（usable_label_ratio > 0），embedding re-rank 就能落地。3 ASR segments 输入就能上出 75% 有效 label——我的“基数效应”初判在 G1 v3 有误。

4. **已知 limitations**。
   - G2_ml（西虹市首富，4.5 GB）未上：1200s 不足以走完 detect_scenes 600s+ + match/W transcribe + faster-whisper fallback，超出 timeout 被杀。
   - `align_status: failed` 不是 bug 而是设计预期（faster-whisper segment-level only），详细状态语义表参见 [`docs/ARCHITECTURE.md`](../ARCHITECTURE.md#status-align-语义)。
   - G3_ml 总耗时 59min 表明大文件（1.6 GB 飞驰人生 3）下场景检测与 transcript 转录是占大头。
