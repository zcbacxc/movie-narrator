## 关键发现

### ✅ PR #72 whitelist 修复**确实生效**

`diversity.max_reuse: 1` 与 `prompt_max_chars_per_sentence: 3` 都到达 `ctx.metadata`。这是 schema → merge → load → runner 链路上 12 params 全部 re-aligned 的**端到端证据**。

### ✅ PR #71 audit schema 设计正确

`script_truncated.details` 18 项真实记录 + `diversity.swaps_log` schema 准备就绪（即使 swaps=0 是数据不克触发的问题，不是 schema 问题），LLM 出口有数据时 audit 字段能不丢信号。

### ⚠️ diversity.swaps 真实 = 0 的原因

飞驰人生3.mp4 充裕——`match_summary.scenes_after_drop = 2424` 个 scene。18 个 narration segment 各 embedding 匹配到不重叠 scene，**自然 diversity 100%**。滑窗 3 内的 max_reuse ≤ 1 永远不会被超过，**swap 没有机会触发**。

这**不是代码 bug**——为了从 audit 角度真触发 swap，需要：
- 更低 scene 池（< 50 个 scenes）+ 更多 narration segments，或
- 同一段视频**重复跑多轮**让 scene 池排空，或
- 加一条对 narration-anchor 的多样化（而不是 scene 锚定）

如果需要 audit-触发 view，可另开 PR 加 `force_diversity_swap_for_test=True` 等媒体参数；当前不做（YAGNI）。
