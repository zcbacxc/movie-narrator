# v0.3 Workflow Config — Design Spec

**Date:** 2026-07-12  
**Status:** Approved for implementation  
**Version target:** `0.3.0`  
**Scope:** ROADMAP v0.3.x — Workflow DSL for pipeline customization + YAML-based pipeline configuration (declarative config only). Web UI and multi-language subtitles remain out of scope.

## Goals

- Let users drive a single job via `mn create --config job.yaml`: movie inputs, soft-step enable/disable, and a small set of step parameters.
- Keep the v0.2 architecture: shared mutable `Context`, fixed linear `STEPS`, pure `(ctx) -> ctx` steps, soft/hard split, `PipelineStatus`.
- Zero-config path must behave like v0.2.2 (existing tests stay green).
- Clear, early errors for bad YAML / unknown keys — fail before the pipeline starts.

## Non-goals (explicitly out of v0.3)

- Step reordering, custom step graphs, named profiles
- Full DSL (`when`, retry, parallel, branching)
- Web UI (Gradio / FastAPI)
- Multi-language subtitles / script i18n
- Plugin system / Python SDK (v0.4)
- `mn config validate` subcommand or published JSON Schema
- Putting LLM credentials / endpoints in job YAML (stay on `.env` / `MN_*`)
- Auto-loading `~/.movie-narrator/pipeline.yaml` or project-level defaults (explicit `--config` only)
- Turning off hard steps / partial runs that skip render

## Success criteria

Conformance language: **must** = acceptance requirement; **should** = recommended; **may** = optional.

1. **Backward-compatible CLI:** without `--config`, behavior **must** match v0.2.2 (including required `--movie` and existing flag semantics). On success the CLI **must** still print the final video path.
2. **`--config` path:** `mn create --config job.yaml` **must** run the pipeline with merged settings; if YAML provides `movie`, `--movie` **may** be omitted; if neither CLI nor YAML provides `movie`, exit non-zero with a clear error.
3. **CLI wins:** an explicitly provided CLI flag **must** override the same field from YAML.
4. **Soft-step control:** YAML `steps.<name>: false` **must** set the corresponding `PipelineStatus` field to `disabled` and skip side effects of that step.
5. **Invalid config:** bad YAML, unsupported `version`, unknown keys, or type errors **must** fail before any pipeline step runs, with a readable message and non-zero exit.
6. **Version:** package version **must** be `0.3.0`; ROADMAP Workflow DSL + YAML items **must** be marked done; Web UI and multi-language remain open.
7. **Docs:** README **should** document `--config` and ship an example job file.

## §1 Architecture

### Approach: Config merge layer (not a workflow engine)

v0.3 adds a **declarative job config** in front of the existing runner. It does **not** replace `STEPS`, introduce a DAG, or register plugins.

```
CLI flags + optional job.yaml
        │
        ▼
  load_job_config (YAML → JobConfig)
        │
        ▼
  merge_job (CLI > YAML > Settings > defaults → ResolvedJob)
        │
        ▼
  run_pipeline(...)   # unchanged step order
        │
        ▼
  soft steps read metadata["workflow_steps"] / params
```

### Module layout

| Path | Responsibility |
|------|----------------|
| `src/movie_narrator/workflow/__init__.py` | Public exports: `load_job_config`, `merge_job`, `JobConfigError`, models as needed |
| `src/movie_narrator/workflow/schema.py` | Pydantic `JobConfig` / `JobSteps` / `JobParams`; whitelist validation |
| `src/movie_narrator/workflow/load.py` | Read file, parse YAML, resolve relative paths against the config file directory |
| `src/movie_narrator/workflow/merge.py` | Merge CLI snapshot + `JobConfig` + `Settings` → `ResolvedJob` |
| `src/movie_narrator/workflow/errors.py` | `JobConfigError` (load/validate only; not `PipelineStrictError`) |
| `src/movie_narrator/cli.py` | Add `--config`; conditional `--movie`; call merge then `run_pipeline` |
| `src/movie_narrator/pipeline/runner.py` | Accept optional `workflow_steps` / `params` / `config_path`; write them into `ctx.metadata` |
| Soft steps | Honor `metadata["workflow_steps"]` at entry |
| `detect_scenes` / `match_clips` / `research_plot` | Prefer metadata params, then `Settings` |
| `pyproject.toml` | Add `pyyaml` as a core dependency |
| `examples/job.example.yaml` | Documented sample |
| Docs | ROADMAP, ARCHITECTURE, README |

### What stays fixed

- `STEPS` order and membership (13 steps as of v0.2.2)
- Hard vs soft classification and `SOFT_STATUS_STEPS`
- `PipelineStatus` field set: `research | align | scene | match | bgm | export`
- Debug CLI commands
- Stdout contract: print `video_path` on success

### Dependency

- **PyYAML** (`pyyaml`) — core dependency (not an extra). Sufficient for load + error line info; no need for ruamel round-trip.

## §2 YAML schema

### 2.1 Example

```yaml
# job.yaml — every field optional; omit = keep Settings / CLI defaults
version: 1

movie: 飞驰人生
style: 热血搞笑
duration: 60
voice: zh-CN-YunxiNeural
format: "16:9"
keep_cache: false

# Paths: relative paths resolve against the directory containing this file
video: ./films/飞驰人生.mp4
library_dir: D:/movies
bgm: ./assets/bgm.mp3
no_bgm: false
no_clips: false
strict: false

# Soft-step toggles (keys align with PipelineStatus fields)
steps:
  research: true
  align: false
  scene: true
  match: true
  bgm: true
  export: false

# Fine-grained parameters (whitelist)
params:
  scene_threshold: 27.0
  match_min_score: 0.25
  research_provider: llm
```

### 2.2 Top-level whitelist

| Key | Type | Notes |
|-----|------|-------|
| `version` | int | Default `1` if omitted; only `1` is accepted |
| `movie` | str | |
| `style` | str | |
| `duration` | int | Must be `> 0` |
| `voice` | str | |
| `format` | str | Exactly `"16:9"` or `"9:16"` |
| `keep_cache` | bool | |
| `video` | str (path) | |
| `library_dir` | str (path) | |
| `bgm` | str (path) | |
| `no_bgm` | bool | |
| `no_clips` | bool | |
| `strict` | bool | |
| `steps` | object | Only the six soft keys |
| `params` | object | Only the keys in §2.3 |

- Unknown top-level keys → validation error (strict whitelist; no silent ignore).
- Root node must be a mapping.

### 2.3 `steps` and `params`

**`steps` allowed keys** (all bool): `research`, `align`, `scene`, `match`, `bgm`, `export`.

- Hard steps **must not** appear under `steps`. Writing e.g. `steps.render: false` is an unknown-key error.
- Only keys **explicitly present** in YAML are recorded in `workflow_steps`. Absent key ≠ `false`.

**`params` allowed keys (v0.3):**

| Key | Type | Consumed by |
|-----|------|-------------|
| `scene_threshold` | float | `detect_scenes` via `metadata`, else `Settings.scene_threshold` |
| `match_min_score` | float | `match_clips` via `metadata`, else `Settings.match_min_score` |
| `research_provider` | str | `research_plot` via `metadata`, else `Settings.research_provider` |

Unknown `params` keys → validation error.

### 2.4 Path resolution

- For `video`, `bgm`, and `library_dir`: if the value is relative, resolve it against **`Path(config_path).parent`** to an absolute path before merge output is used.
- Missing `video`/`bgm` files are **not** hard-failed inside `load_job_config`. After merge, `cli.create` applies the same existence check already used for `--video`.

## §3 Merge rules

### 3.1 Priority (high → low)

```
1. Explicit CLI flags
2. job.yaml
3. Settings / .env (MN_*)
4. Built-in code defaults
```

### 3.2 What counts as “explicit CLI”

| Flag | Sentinel when unset | Explicit when |
|------|---------------------|---------------|
| `--movie` | With `--config`, default `None`; without `--config`, still required | `is not None` |
| `--research / --no-research` | `None` | `is not None` |
| `--video`, `--bgm`, `--library-dir`, `--voice` | `None` | `is not None` |
| `--no-bgm`, `--no-clips`, `--strict`, `--keep-cache` | `False` | **Only `True`** is explicit (`False` may be the Typer default) |

**Options with non-empty Typer defaults** (`style` default `热血搞笑`, `duration` default `60`, `format` default `16:9`):

- If `--config` is set **and** YAML defines the field **and** the CLI value still equals the Typer default → **use YAML**.
- If the CLI value differs from the Typer default → **use CLI**.
- If there is no `--config` → identical to v0.2.2.

### 3.3 Soft-step resolution

| Step field | Resolution |
|------------|------------|
| `research` | CLI `--research/--no-research` if not `None`; else YAML `steps.research` if present; else `settings.research_enabled` |
| `export` | CLI `--no-clips` if `True` forces off; else YAML `steps.export` if present; else `settings.export_clips_default` |
| `bgm` (step) | Existing path/`no_bgm` logic; additionally YAML `steps.bgm: false` forces disabled |
| `align` / `scene` / `match` | No dedicated CLI flag; YAML only; if absent, keep current step-internal logic (deps + inputs) |

Each soft step **must** check at entry:

```python
if ctx.metadata.get("workflow_steps", {}).get("<field>") is False:
    ctx.status.<field> = "disabled"
    # existing disabled logging if any
    return ctx
```

`workflow_steps` contains **only YAML-explicit keys**.

### 3.4 `ResolvedJob`

Internal merge result (not necessarily exported as a public SDK type in v0.3):

```python
class ResolvedJob(BaseModel):
    movie: str
    style: str
    duration: int
    voice: Optional[str]
    format: str
    keep_cache: bool
    video: Optional[str]
    library_dir: Optional[str]
    bgm: Optional[str]
    no_bgm: bool
    no_clips: bool
    strict: bool
    research: Optional[bool]          # None → runner uses settings default path
    workflow_steps: Dict[str, bool]   # YAML-explicit only
    params: Dict[str, Any]            # validated subset
    config_path: Optional[str]        # for metadata traceability
```

`cli.create` flow:

1. Parse Typer args (including optional `--config`).
2. If `config_path`: `job = load_job_config(path)` else `job = None`.
3. `resolved = merge_job(cli_snapshot, job, settings)`.
4. Validate `resolved.movie` present; validate `resolved.video` file if set.
5. `run_pipeline(**pipeline_kwargs_from(resolved))`.

### 3.5 `run_pipeline` surface

Keep existing keyword arguments. Add optional:

- `workflow_steps: Optional[Dict[str, bool]] = None`
- `params: Optional[Dict[str, Any]] = None`
- `config_path: Optional[str] = None`

When all three are default/empty, metadata keys for workflow **must** be omitted or empty such that soft steps behave as in v0.2.2.

Runner writes into `ctx.metadata` at least:

- existing keys unchanged in meaning
- `workflow_steps` (if non-empty)
- each provided param key (`scene_threshold`, `match_min_score`, `research_provider`)
- `config_path` when set

Steps that currently read thresholds only from `get_settings()` **must** be updated to:

```python
threshold = ctx.metadata.get("scene_threshold", settings.scene_threshold)
```

(and the same pattern for the other params).

## §4 CLI contract

```bash
mn create --config job.yaml
mn create --config job.yaml --movie OtherTitle   # CLI overrides YAML movie
mn create --movie 飞驰人生                       # no config → v0.2.2 behavior
```

- New option: `--config PATH` (`Optional[str] = None`).
- With `--config`: `--movie` is not required by Typer; post-parse validation requires movie from CLI or YAML.
- Without `--config`: `--movie` remains required.
- Success stdout: still the final `video_path` (primary user-visible contract).

Implementation requirement: single `create` command; `--movie` uses `Optional` default `None` with a post-parse check:

1. If `movie is None` and there is no `--config` → `BadParameter`: movie is required.
2. If `movie is None` and config/merge also lack movie → `BadParameter`: `movie is required (set --movie or config.movie)`.
3. Otherwise proceed with merged movie.

Do **not** keep Typer-required `--movie` (that would block `mn create --config job.yaml` alone).

## §5 Error model

### 5.1 Load / validate (before pipeline)

| Situation | Mechanism | Message shape |
|-----------|-----------|---------------|
| Config path missing | `typer.BadParameter` / exit 2 | `config not found: <path>` |
| YAML syntax error | `JobConfigError` → exit 1 | `invalid YAML: <msg> (line N)` when available |
| Root not a mapping | `JobConfigError` | `job config must be a mapping` |
| Unsupported `version` | `JobConfigError` | `unsupported config version: N (supported: 1)` |
| Unknown key (top / steps / params) | `JobConfigError` | `unknown key: 'foo' (allowed: …)` |
| Type / range errors | `JobConfigError` | e.g. `duration must be int, got str`; `duration must be > 0`; `format must be '16:9' or '9:16'` |
| No movie after merge | `typer.BadParameter` | `movie is required (set --movie or config.movie)` |
| Video path missing after merge | existing `BadParameter` | `video not found: …` |

`JobConfigError` is **not** a subclass of `PipelineStrictError`.

### 5.2 Runtime

Unchanged:

- Soft failure + `strict` → `PipelineStrictError`
- Hard failure → propagate
- Soft disabled via YAML → `status.* = "disabled"`, continue

### 5.3 No validate subcommand

v0.3 does not add `mn config validate`. Unit tests call `load_job_config` / `merge_job` directly.

## §6 Testing strategy

| Group | Coverage |
|-------|----------|
| Load success | Minimal YAML (`movie` only); full document; relative path resolution |
| Load failure | Missing file; corrupt YAML; unknown keys; bad version; bad types / format / duration |
| Merge priority | CLI overrides YAML; YAML overrides Settings; bool sentinel rules; style/duration/format default rule with `--config` |
| Steps semantics | `steps.research: false` → `status.research == "disabled"` and research side effects not invoked (mock) |
| Params semantics | `scene_threshold` (etc.) land in metadata and are read by the step (mock settings) |
| No-config regression | Existing CLI / runner / soft-step tests remain green |
| CLI | Typer `CliRunner` + mocked `run_pipeline` asserts kwargs for `--config` and override cases |

Out of unit-test scope: real LLM / Edge-TTS / MoviePy. Optional later: CI smoke with `CI=1` and an example YAML.

## Design decisions (summary)

| Decision | Rationale |
|----------|-----------|
| Config merge layer, not step registry / DAG | Matches “toggle + params only”; YAGNI vs v0.4 plugins |
| Strict key whitelist | Typos must not silently no-op |
| CLI > YAML > Settings | Predictable overrides; scripts can pin flags |
| Relative paths vs config file dir | Portable job files |
| No LLM settings in YAML | Avoid secrets in project-shared job files |
| Explicit `--config` only | No surprising auto-load side effects |
| `workflow_steps` only explicit keys | Distinguish “unset” from `false` |
| Hard steps not configurable | Success criteria reject partial-run scope |
